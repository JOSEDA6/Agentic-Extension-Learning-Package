0. 先聊聊：Agent 到底是什么？

你用过 ChatGPT 吗？你问它「今天天气怎么样」，它回答「抱歉，我无法获取实时信息」。这是因为单纯的 ChatGPT 只有一个嘴巴——它能说话，但不能动手。

而 AI Agent（智能体），就是给这个「大脑」装上手和脚。它不仅能思考，还能：查天气 API、计算数学题、读取文件、发邮件……

把大模型想象成一个被困在房间里、只靠对讲机与外界沟通的天才。Agent 就是你给这个天才配了一群小助手，每个小助手说：「老板，我能帮你查天气」「老板，我能帮你算数学」。天才通过对讲机喊：「计算！3+5」，小助手算出 8 回报给天才，天才接着说：「等于 8」。

今天我们就要写出这样一个 Agent：一个能自己调用计算器来回答数学问题的 Python 程序。整个过程只需要 约 50 行代码，你复制粘贴就能跑。

🧠 核心概念速览 一个最简 Agent 只做三件事：
① 收到你的问题 → ② 把问题发给大模型（大脑）→ ③ 大模型说「我需要算个数」，Agent 去算完把结果塞回去 → ④ 大模型给出最终答案。
这就是所谓的 「调用—反馈」循环。

1. 第一步：安装 Python（你的机器人引擎）

Python 是一门编程语言，你可以把它理解成「机器人的发动机」。没有它，代码只是一堆文字，跑不起来。

1.1 检查你的电脑是否已经装了 Python

按键盘上的 Win + R，输入 cmd 然后回车，会弹出一个黑乎乎的窗口——这叫终端（也叫命令行）。在里头输入：

shell 
python --version

如果出现类似 Python 3.10.x 或 Python 3.11.x 的字样，说明你已经装了，跳过下一步。

1.2 如果没装：下载安装

用浏览器打开：https://www.python.org/downloads/

点击黄色大按钮下载。下载完成后双击打开，务必勾选底部的 "Add Python to PATH"，然后点 "Install Now"，一路下一步直到完成。

⚠️ 注意！「Add Python to PATH」一定要勾上 如果不勾，后面在终端里输入 python 会提示「找不到命令」，你得重新安装一遍。

装完后，关掉终端重新打开，再输一次 python --version，看到版本号就成功了。

2. 第二步：安装 openai 库（机器人的"翻译官"）

Python 本身只会执行基础命令。要让 Python 能跟 OpenAI 的服务器对话，需要一个翻译官——在编程里，这叫库（library）。

打开终端（Win + R → 输入 cmd → 回车），输入下面这行命令然后回车：

shell 
pip install openai

你会看到一堆文字滚过去，最后出现 Successfully installed openai-x.x.x 就代表装好了。

💡 pip 是什么？ pip 是 Python 的「应用商店」。你需要什么功能，就用 pip install 名字 来下载安装。openai 就是我们今天唯一的依赖。
⚠️ 如果提示「pip 不是内部命令」 说明你装 Python 时没勾「Add Python to PATH」。去重新运行 Python 安装程序，选「Modify」，把那个勾打上。

4. 第四步：写出 Agent 的全部代码

好，现在是最激动人心的部分。我们要亲手写出约 50 行代码，创造一个能自己「思考 + 动手」的机器人。

4.1 在桌面新建一个文件夹

在你的电脑桌面上，右键 → 新建 → 文件夹，命名为 my-first-agent。

4.2 新建代码文件

打开记事本（按 Win 键，输入「记事本」回车）。把下面的代码完整复制粘贴进去：

Python — agent.py（完整代码，约50行） 
#!/usr/bin/env python3
"""my_first_agent.py —— 你的第一个 AI Agent。"""

import json
from openai import OpenAI

# ── 1. 设置钥匙和地址 ───────────────────────────
client = OpenAI(
    api_key="{{YOUR_API_KEY}}",          # ← 替换成你的 Key
    base_url="https://api.openai.com/v1"  # 国产模型改这里
)

# ── 2. 给 Agent 配备工具箱 ──────────────────────
TOOLS = [{
    "type": "function",
    "function": {
        "name": "calculator",
        "description": "计算数学表达式，支持加减乘除和括号。例如: '2+3*4' 返回 14",
        "parameters": {
            "type": "object",
            "properties": {
                "expression": {
                    "type": "string",
                    "description": "要计算的数学表达式，例如 '15*23+100'"
                }
            },
            "required": ["expression"]
        }
    }
}]

# ── 3. 执行工具的 "手" ─────────────────────────
def execute_tool(name, args):
    """Agent 的手 —— 真正去干活。"""
    if name == "calculator":
        return str(eval(args["expression"]))
    return f"未知工具: {name}"

# ── 4. Agent 的大脑 + 循环 ─────────────────────
def agent(question):
    """核心循环：问问题 → 大模型思考 → 需要工具就调用 → 再思考 → 给出答案"""
    messages = [
        {"role": "system", "content": "你是一个有用的助手。遇到数学计算时，必须使用 calculator 工具。"},
        {"role": "user",   "content": question}
    ]

    while True:   # 循环，直到大模型不再要求工具
        response = client.chat.completions.create(
            model="gpt-4o-mini",       # 便宜又好用的模型
            messages=messages,
            tools=TOOLS
        )
        msg = response.choices[0].message

        if msg.tool_calls:
            # 大模型说 "我要用工具！" —— 帮它执行
            messages.append(msg)   # 把大模型的 "工具请求" 记入对话
            for tc in msg.tool_calls:
                name = tc.function.name
                args = json.loads(tc.function.arguments)
                result = execute_tool(name, args)
                messages.append({
                    "role": "tool",
                    "tool_call_id": tc.id,
                    "content": result
                })
                print(f"🔧 调用工具 {name}({args}) → 结果: {result}")
        else:
            # 大模型直接给出了答案 —— 返回！
            return msg.content

# ── 5. 启动！───────────────────────────────────
if __name__ == "__main__":
    print("🤖 Agent 启动！输入你的问题（输入 quit 退出）\n")
    while True:
        q = input("👤 你: ").strip()
        if q.lower() == "quit":
            break
        answer = agent(q)
        print(f"🤖 Agent: {answer}\n")

5. 第五步：逐行拆解 —— 每一行在干什么？

复制粘贴跑起来固然爽，但理解每一行为什么这么写，你才能从「复制侠」变成「创造者」。下面把整段代码拆成 5 块来讲解。

第 1 块：导入依赖

import json
from openai import OpenAI

json 是 Python 自带的，用来解析工具参数。openai 是我们刚才用 pip 安装的翻译官。

这就像你造机器人之前，先把需要的零件摆在桌上。json 是一把螺丝刀（Python 自带），openai 是一个特殊的发动机（需要额外下载）。

第 2 块：配置钥匙

client = OpenAI(
    api_key="sk-your-key-here",
    base_url="https://api.openai.com/v1"
)

client 就是你和 OpenAI 服务器之间的专线电话。api_key 是密码，base_url 是电话号码（服务器地址）。

第 3 块：定义工具（说明书）

TOOLS = [{
    "type": "function",
    "function": {
        "name": "calculator",
        "description": "计算数学表达式...",
        "parameters": { ... }
    }
}]

这是一个说明书，告诉大模型：「嘿，我这有一个计算器可以用。你如果想算数，就按这个格式喊我。」大模型读完说明书，就知道什么时候该调用工具、参数怎么填。

你给助手写了一本「技能手册」：第 1 页写着「计算器 —— 用法：把算式给我，我帮你算」。助手读了手册，遇到数学题就知道翻到第 1 页。

第 4 块：执行工具（手）

def execute_tool(name, args):
    if name == "calculator":
        return str(eval(args["expression"]))

当大模型说「我要用 calculator」，这行代码就真正去算。eval() 是 Python 自带的一个函数，能把字符串 "3+5" 算成数字 8。

第 5 块：核心循环（灵魂所在）

这是整个 Agent 最关键的逻辑。以下面的流程图来看：

while True:
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
        tools=TOOLS            # ← 把 "技能手册" 发给大模型
    )
    msg = response.choices[0].message

    if msg.tool_calls:         # 大模型要求调工具
        # ... 执行工具，把结果塞回 messages ...
    else:                      # 大模型直接给答案
        return msg.content

这个过程画成图是这样的：

👤 你问问题
⬇
🧠 大模型收到问题 + 技能手册
⬇
❓ 大模型判断：需要工具吗？
╱                      ╲
需要               不需要
⬇                        ⬇
🔧 执行工具            💬 直接回答
⬇
📩 把结果塞回对话
⬇
🔄 再问大模型（回到开头）

这个循环会一直转，直到大模型觉得「信息够了，我可以回答了」，然后 return 把答案交出来。

🧠 这个循环模式来自 claw-code-main 的 query_engine.py 在 claw-code-main 中，你可以看到完全一样的模式：submit_message() 收到用户输入 → 匹配工具 → 执行工具 → 返回结果。只不过它更复杂（有 token 预算、会话持久化、权限检查等）。我们保留了最核心的骨架，去掉了所有「暂时用不上的肌肉」。

7. 第七步：常见报错 & 解决办法

新手写代码，报错是家常便饭。看到红色错误信息不要慌——报错是 Python 在帮你找问题，它告诉你的每一行都是线索。

① ModuleNotFoundError: No module named 'openai'

意思：「我找不到 openai 这个库」

解决：你没装 openai。终端里执行 pip install openai

② SyntaxError: invalid syntax

意思：「你写的某一行语法不对，我看不懂」

解决：最常见的原因是少写了冒号 :、括号没配对、缩进不对。检查错误提示里指出的那一行。

③ IndentationError: expected an indented block

意思：「你说要开始一个代码块（比如 if 或 def 后面），但我没看到缩进」

解决：在 Python 里，if、def、while 等语句的冒号后面，下一行必须用 4 个空格缩进。检查你是不是漏了缩进。

④ AuthenticationError: Incorrect API key provided

意思：「你的 API Key 不对」

解决：检查你是不是把 {{YOUR_API_KEY}} 替换成真实的 Key 了？有没有多复制空格？Key 是否已过期或被删除了？

⑤ RateLimitError: You exceeded your current quota

意思：「你的账户余额不够了」

解决：去 OpenAI 后台充值。或者切换到更便宜的模型（比如把 gpt-4o-mini 改成 gpt-3.5-turbo）。

⑥ 终端里输入 python 后提示「不是内部或外部命令」

解决：重装 Python，务必勾选「Add Python to PATH」。或者搜「Windows 添加 Python 到环境变量」。

💡 遇到上面没列出的报错怎么办？ 把错误信息复制下来，去 Google 搜或者问 ChatGPT。90% 的情况下，你都不是第一个遇到这个问题的人。

9. 附录：如何看懂 claw-code-main 这种大项目？

现在你已经亲手写过一个 Agent 了。回头看 claw-code-main 这个庞大的项目时，你可能会觉得「哇，好复杂」。别怕，下面教你怎么用一个简单的思路看穿它：

9.1 核心思路：找「三件套」

任何一个 Agent 项目，你只需要找到三个东西：

要找什么在我们教程里对应在 claw-code-main 里对应
LLM 调用（大脑）client.chat.completions.create(...)query_engine.py 中的 submit_message()
工具定义（说明书）TOOLS = [{...}]tools.py 中的 PORTED_TOOLS
主循环（心跳）while True: ...runtime.py 中的 run_turn_loop()

9.2「删除法」理解代码

看一个复杂文件时，问自己：如果我把这段删掉，Agent 还能运行吗？

  • 如果能 —— 那这段就是辅助代码（日志、美化输出、缓存等），可以先跳过不看
  • 如果不能 —— 那这段就是核心逻辑，需要仔细理解

用这个方法看 query_engine.py：compact_messages_if_needed()（消息压缩）可以删，但 submit_message()（提交消息）绝对不能删。

9.3 大项目 = 核心循环 + N 层包装

claw-code-main 看起来很庞大，但本质上是：

我们的 50 行 Agent
＋ 命令路由（把用户输入匹配到合适的工具）
＋ Token 预算管理（防止花太多钱）
＋ 会话持久化（把聊天记录存到硬盘）
＋ 权限系统（防止危险操作）
＋ CLI 界面（命令行参数解析）
＝ claw-code-main

你不需要第一天就看懂全部。先把核心三件套吃透，然后每次只加一个「包装层」来理解，循序渐进。
